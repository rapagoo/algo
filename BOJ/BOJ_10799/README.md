# BOJ 10799 - 쇠막대기

## 1. 핵심 아이디어
- 괄호 문자열을 한 번 순회하면서 `()`는 레이저, 그 외 `)`는 막대의 끝으로 구분한다.
- `count`를 “현재 열려 있는 막대의 수(스택 크기)”처럼 유지한다.
- 레이저를 만나면 현재 열려 있는 막대 수만큼 조각이 추가되고, 막대 끝을 만나면 1조각이 추가된다.

## 2. 불변식 (Invariant)
- 현재 위치까지 처리했을 때 `count`는 아직 닫히지 않은 `'('`의 개수(= 진행 중인 막대 개수)이다.

## 3. 시간 복잡도
- 시간: O(n)
- 공간: O(1)

## 4. 내가 한 실수 / 주의할 점
- 레이저 `()` 처리에서 `count--`를 먼저 하지 않으면 `ans += count`가 1 크게 더해져 오답이 된다.
- `prev`로 직전 문자만 보고 레이저를 판별하므로, 입력이 공백 없이 괄호로만 주어진다는 조건에 의존한다.
- `count`가 음수가 되지 않도록(잘못된 괄호 입력이 없다는 전제) 문제 조건을 확인한다.

## 5. 복습 질문 (Active Recall)
1. `count`가 의미하는 값을 한 문장으로 정의해봐라.
2. 레이저 `()`에서 왜 `ans += count`이고, 왜 그 전에 `count--`를 하는가?
3. 막대의 끝 `)`에서 왜 `ans += 1`인가?
4. `"()"`, `"(())"`, `"(((())))"`를 직접 추적하며 `count/ans` 변화를 설명해봐라.
