# BOJ 9012 - 괄호

## 1. 핵심 아이디어
- '('를 만나면 count++, ')'를 만나면 count--
- 순회 중 count가 음수가 되는 순간 접두사 조건 위반 → 즉시 실패
- 끝까지 봤을 때 count==0 이면 VPS

## 2. 불변식 (Invariant)
- i번째 문자까지 처리했을 때 count = (지금까지의 '(' 개수) - (지금까지의 ')' 개수) 이며, VPS라면 모든 접두사에서 count < 0 이 되지 않는다.

## 3. 시간 복잡도
- 시간: O(문자열 길이) (각 테스트케이스)
- 공간: O(1)

## 4. 내가 한 실수 / 주의할 점
- ')' 처리 중 count가 음수면 즉시 break 해야 함(접두사 조건)
- 마지막에 count==0인지 반드시 확인해야 함(전체 균형 조건)
- 입력이 '('와 ')'만 온다는 문제 조건에 의존(일반화 시 예외 처리 필요)
