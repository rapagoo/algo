# BOJ 4949 - 균형잡힌 세상

## 1. 핵심 아이디어
- 한 줄씩 입력을 받아 `.` 한 줄이 나오면 종료한다.
- 여는 괄호 `(`, `[`는 스택에 push 한다.
- 닫는 괄호 `)`, `]`는 스택 top이 대응되는 여는 괄호인지 즉시 확인하고, 아니면 실패 처리한다.

## 2. 불변식 (Invariant)
- 현재까지 처리한 문자열의 접두사에서, 스택에는 “아직 닫히지 않은 여는 괄호들”만 올바른 중첩 순서로 들어 있다.

## 3. 시간 복잡도
- 시간: O(L) (각 줄 길이 L에 대해 한 번 순회)
- 공간: O(L) (최악의 중첩 깊이)

## 4. 내가 한 실수 / 주의할 점
- “마지막에 스택이 비었으면 yes”로만 판정하면 오답이 날 수 있다(예: `()]`, `()]`는 중간에 불일치가 발생).
- 닫는 괄호를 만나는 순간 `st.empty()` 및 `st.top()`이 올바른 타입인지 즉시 검증해야 한다.
- 문자열에는 다른 문자도 섞여 있으므로 괄호 문자인 경우에만 스택 로직을 적용한다.

## 5. 복습 질문 (Active Recall)
1. `()]`가 왜 마지막에 스택이 비어도 오답이 되는지 처리 흐름으로 설명해라.
2. `([)]`가 오답인 이유를 “top 검사” 관점에서 설명해라.
3. 닫는 괄호를 처리할 때 반드시 확인해야 하는 조건 2가지는 무엇인가?
4. 한 줄 처리가 끝난 뒤 최종적으로 yes/no를 결정하는 조건은 무엇인가?
