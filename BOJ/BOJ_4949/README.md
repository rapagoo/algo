# BOJ 3986 - 좋은 단어

## 1. 핵심 아이디어
- 문자열을 왼쪽부터 읽으며 스택으로 인접한 같은 문자 쌍을 제거한다.
- 현재 문자와 스택 top이 같으면 pop, 다르면 push 한다.
- 모든 문자를 처리한 뒤 스택이 비어 있으면 “좋은 단어”로 판정한다.

## 2. 불변식 (Invariant)
- i번째 문자까지 처리한 뒤 스택에는 s[0..i]에서 “인접한 같은 문자 쌍을 제거” 규칙을 가능한 한 적용한 결과(축약 문자열)가 그대로 들어 있다.

## 3. 시간 복잡도
- 시간: O(Σ|s|) (각 문자는 최대 1회 push, 1회 pop)
- 공간: O(|s|) (문자열 1개 처리 시 최악)

## 4. 내가 한 실수 / 주의할 점
- 인덱스 기반 반복에서 signed/unsigned 혼동 가능 → range-for로 안전하게 처리
- 스택 비어있을 때 top 접근 금지(반드시 empty 체크)
- 마지막에 스택이 비었는지로만 판정(중간 상태로 성급히 결론 내지 않기)

## 5. 복습 질문 (Active Recall)
1. 이 문제에서 스택이 표현하는 상태(불변식)는 무엇인가?
2. top과 현재 문자가 같을 때 pop 하는 규칙이 문제의 “쌍 제거”와 정확히 대응되는 이유는?
3. ABBA 같은 입력에서 스택 변화 과정을 단계별로 설명해보라.
4. 왜 최종 스택이 비어야만 “좋은 단어”인가?
